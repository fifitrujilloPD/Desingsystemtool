# Design System Tool

This project is a Design System tool built to function as a visual component library and internal reference for reusable UI components. It was created to support the design area with a more structured workflow and to optimize development time when introducing updates, scaling the system, or creating new implementations.

The system is structured using the **Atomic Design** methodology, making it easier to organize components consistently and scale the library over time.

## Project objective

The main objective of this tool is to bridge design and development through a structured component environment that improves visibility, consistency, and speed across the product workflow.

## Development approach

The creation of this project was AI-assisted.

After defining the initial components, I used **Figma Make** to generate the first stable version of the project, including its base structure and layout. Once this initial foundation was ready, I uploaded the project to **GitHub** and continued the implementation in **Cursor**, where I iterated and created additional components using the **Figma MCP** workflow.

## Why use Figma Make first?

One of the biggest challenges when starting a project from the design side is creating a solid initial structure that is stable enough to continue iterating on. Figma Make helped solve this by providing a reliable base with minimal structural errors.

With that foundation already established, Cursor became the ideal environment to continue refining the project and expanding the component library efficiently.

## Methodology

This project follows the **Atomic Design** methodology, organizing the system into scalable layers:

- Atoms
- Molecules
- Organisms
- Templates
- Pages

This approach helps keep the Design System modular, reusable, and easier to maintain over time.

## Workflow summary

1. Initial component definition
2. Base generation with Figma Make
3. Project upload to GitHub
4. Iteration and refinement in Cursor
5. Component creation through Figma MCP

## Running the code

Run `npm i` to install the dependencies.

Run `npm run dev` to start the development server.
